# Homelab Docker Stack — Raspberry Pi Setup

Redistributable IoT/home automation stack: **Home Assistant** + **Mosquitto MQTT** + **Portainer**.

Designed as a reusable template: clone it, edit `.env`, run `setup.sh`, and you have a
working home automation instance with MQTT devices, climate controls, dashboards, and
weather pre-configured.

---

## Prerequisites

- Raspberry Pi (aarch64) with Raspberry Pi OS or similar Debian-based distro
- Internet access for pulling Docker images
- (Optional) IoT devices publishing to MQTT on topics prefixed `kk/` or `sai-nivas/`

---

## Quick Start

```bash
# 1. Clone / copy this repo to your Pi
git clone <your-repo-url> ~/docker
cd ~/docker

# 2. Create your environment config
cp .env.example .env
nano .env    # Edit all values for your setup (location, MQTT prefixes, etc.)

# 3. Run the bootstrap script (installs Docker, creates dirs, deploys config)
chmod +x setup.sh
./setup.sh

# 4. Log out and back in (if Docker was just installed, for group permissions)

# 5. Create the Mosquitto MQTT user/password
docker run --rm -it \
  -v $DOCKER_BASE/mosquitto/config:/mosquitto/config \
  eclipse-mosquitto:2.0.21 mosquitto_passwd -c /mosquitto/config/pwfile aru-mqtt
#   ^ Enter your desired MQTT password when prompted

# 6. Start the stack
docker compose up -d

# 7. Access Home Assistant and complete first-boot setup
#    http://<pi-ip>:8123  (login: aru_ha / ***REMOVED*** — CHANGE THIS IMMEDIATELY)
```

See **Post-Deploy Checklist** below for required first-boot configuration.

## What `setup.sh` Does

1. **Installs Docker and Docker Compose plugin** (if not present)
2. **Creates the directory structure** under `$DOCKER_BASE`
3. **Generates `secrets.yaml`** for Home Assistant from `.env` values
4. **Copies config files** — including the seeded `.storage/` (integrations, dashboard, entities) — and applies MQTT topic prefix substitution to entity YAML + `.storage/lovelace` + `.storage/core.config_entries`
5. **Applies mains voltage substitution** for energy calculation templates
6. **Sets Mosquitto directory ownership** (UID 1883, required by the broker)
7. **Generates self-signed TLS certificates** for Portainer

```
│   ├── configuration.yaml        # Main HA config (uses !secret refs)
│   ├── secrets.yaml              # Generated by setup.sh (gitignored)
│   ├── integrations/             # HA package includes (automation, mqtt, sensor, etc.)
│   ├── entities/                 # MQTT sensors, switches, climate, binary sensors
│   └── .storage/                 # Seeded HA state (integrations, dashboard, registry)
│       ├── core.config_entries   # MQTT broker, Tasmota, Open-Meteo, mobile_app, etc.
│       ├── core.entity_registry  # 430 pre-registered entities
│       ├── lovelace              # Dashboard YAML
│       ├── mobile_app            # Mobile app registrations
│       ├── person                # Person tracking
│       └── ... (see "Seeded .storage" below)

    └── mosquitto.conf            # Broker configuration
```

**On the target Pi after `setup.sh` runs:**

```

├── hassio/homeassistant/         # Full HA config (with generated secrets.yaml + seeded .storage)
├── mosquitto/

│   ├── data/                     # Persistent MQTT data (gitignored)

    └── portainer.db              # Created on first run (gitignored)
```

## Seeded `.storage/` (Pre-configured Home Assistant)

The `.storage/` directory is **included in the template** so new instances start with a
working baseline — you don't start from a blank HA install.

### What's seeded (shipped with the template)

| File | Contents |
|------|----------|
| `core.config_entries` | Integration configs: MQTT broker, Tasmota, Open-Meteo, mobile_app (×3), systemmonitor, rpi_power, google_translate, met |
| `core.entity_registry` | 430 pre-registered entities (AC climates, MQTT sensors/switches, tasmota devices, mobile_app sensors) |
| `core.device_registry` | Device definitions |
| `core.area_registry` | Area definitions |
| `lovelace` | Dashboard YAML (references `weather.arunivas`, AC climates, switches) |
| `lovelace_dashboards` | Dashboard list |
| `mobile_app` | Mobile app integration data |
| `person` | Person tracking config |
| `auth` / `auth_provider.homeassistant` | Default login: **user `aru_ha`, password `***REMOVED***`** |
| `onboarding` | Marks onboarding complete |
| `homeassistant.exposed_entities` | Voice assistant exposure settings |
| `core.config` | Core HA config (name, location, etc.) |
| `http` | HTTP server config |

### What's NOT seeded (auto-regenerated per instance, gitignored)

| File | Why excluded |
|------|--------------|
| `core.uuid` | Unique instance identifier — must be different per install |
| `core.restore_state` | Runtime state cache — regenerated on boot |
| `http.auth` | JWT signing key — security: must be unique per instance |
| `trace.saved_traces` | Debug traces — runtime only |
| `frontend.user_data_*` | Per-user frontend preferences |

## Post-Deploy Checklist (REQUIRED for new instances)

After `docker compose up -d` on a fresh deploy, log into Home Assistant
(`http://<pi-ip>:8123`) and complete these steps:

### 1. Change the default password ⚠️
The seeded login is **`aru_ha` / `***REMOVED***`**. Change it immediately:
- Go to **Profile → Security** (bottom left) → change password

### 2. Update the MQTT broker IP
The seeded MQTT config points to `10.1.0.6` (the original Pi's IP). Update it:
- **Settings → Devices & Services → MQTT → Configure**
- Change the broker IP to the new Pi's IP
- Update the username/password to match the `pwfile` you created in step 5 of Quick Start

### 3. Re-register mobile apps
The seeded `mobile_app` entries (Kumar's iPhone, SM-M346B, Kumaresan's iPad) have
push tokens tied to the old instance. Delete and re-add them:
- **Settings → Devices & Services → Mobile App** → delete old entries
- Install the HA Companion app on each phone and log in to this new instance

### 4. Review weather location
Open-Meteo is pre-configured and free (no API key). It uses `zone.home` for location,
which comes from `HA_LATITUDE`/`HA_LONGITUDE` in `.env`. Verify at:
- **Settings → Devices & Services → Open-Meteo**

### 5. Verify IoT devices are publishing
Check the MQTT broker for device traffic:
```bash
docker exec mosquitto mosquitto_sub -h 127.0.0.1 -u aru-mqtt -P '<your-password>' -t 'kk/#' -v
```
You should see `LWT` and sensor values from your ACs, fans, and lights.

## Managing the Stack

### Start / stop / restart
```bash
cd ~/docker
docker compose up -d        # Start all services
docker compose down          # Stop all services
docker compose restart homeassistant   # Restart just HA
```

### View logs
```bash
docker compose logs -f homeassistant    # Follow HA logs
docker compose logs -f mosquitto        # Follow MQTT broker logs
docker exec mosquitto tail -f /mosquitto/log/saimosquitto.log   # MQTT log file
```

### Home Assistant CLI (inside container)
```bash
docker exec homeassistant python3 -m homeassistant --config /config --script check_config
#   ^ Validates configuration without restarting
```

### Updating Home Assistant
1. Update the image tag in `docker-compose.yml` (e.g. `:2025.4.0`)
2. **Back up first**: in HA UI, **Settings → System → Backups → Create Backup**
3. Pull and restart:
   ```bash
   docker compose pull homeassistant
   docker compose up -d homeassistant
   ```
4. Check logs for errors: `docker compose logs homeassistant`

## Migration from an Existing Pi

If you have an existing HA instance and want to migrate its data to a new Pi:

```bash
# On OLD Pi — back up runtime + integration data
tar czf homelab-backup.tar.gz \
  -C $DOCKER_BASE \
  hassio/homeassistant/.storage \
  hassio/homeassistant/home-assistant_v2.db \
  mosquitto/config/pwfile \
  mosquitto/data \
  portainer/portainer.db

# On NEW Pi — run setup first (creates dirs + deploys template config), then restore
cd ~/docker
./setup.sh
tar xzf homelab-backup.tar.gz -C $DOCKER_BASE
docker compose up -d
```

> **Note:** Restoring the old `.storage/` will overwrite the seeded template config.
> This is intentional — you want your old integrations, dashboard, and entities.

## Notes

- All containers use `restart: always` and survive reboots.
- **MQTT topic prefixes** (`MQTT_PREFIX_1`, `MQTT_PREFIX_2`) must match your IoT device
  firmware. `setup.sh` will substitute these across entity YAML and `.storage/` files
  if you change them from the defaults (`kk`, `sai-nivas`).
- **Portainer** binds `/var/run/docker.sock` for host Docker management.
- **Open-Meteo** (weather) is free and requires no API key or account. It replaced
  AccuWeather, which now requires a paid subscription.
- **Image versions are pinned** in `docker-compose.yml` for reproducibility.
- Secrets and instance-unique files (`.env`, `secrets.yaml`, `pwfile`, `core.uuid`,
  `http.auth`, DBs, certs, keys) are gitignored — see `.gitignore`.
